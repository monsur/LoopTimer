# Publishing Loop Timer to the App Store

This comprehensive guide covers everything needed to publish your app to the Apple App Store.

## 1. Apple Developer Program ($99/year)

### Enrollment
- Enroll at https://developer.apple.com/programs/
- This is **required** for App Store distribution (free accounts can only test on personal devices)
- Takes 24-48 hours for approval
- Renews annually at $99/year

### What You Get
- Ability to publish apps on App Store
- Access to beta iOS releases
- Advanced app capabilities
- App analytics
- TestFlight for beta testing

## 3. Prepare Your App in Xcode

### Update Bundle Identifier & Version

1. Open project settings in Xcode
2. Select LoopTimer target
3. In General tab:
   - **Bundle Identifier**: Set unique ID (e.g., `com.yourname.looptimer`)
   - **Version**: Set to `1.0.0` for first release
   - **Build**: Set to `1`
4. Repeat for LoopTimerWidgetExtension target (use `com.yourname.looptimer.widget`)

### Privacy Manifest

Create or verify privacy declarations:

**Data Collected:** None

**Required Reasons:**
- Background Audio: "Plays timer completion sounds"
- Notifications: "Notifies user of timer events"
- Live Activities: "Shows timer on lock screen"

### Category Selection

Choose primary category:
- **Productivity** (recommended)
- **Utilities** (alternative)
- **Health & Fitness** (if positioning for workouts)

### Age Rating

Complete questionnaire in App Store Connect:
- No violent content
- No mature themes
- No unrestricted web access
- No user-generated content
- Likely rating: **4+**

Before submission, test:

## 4. App Store Connect Setup

### Create App Record

1. Go to https://appstoreconnect.apple.com
2. Click **My Apps** ‚Üí **+** ‚Üí **New App**
3. Fill in:
   - **Platform**: iOS
   - **Name**: Loop Timer (or your chosen name)
   - **Primary Language**: English
   - **Bundle ID**: Select the one from Xcode
   - **SKU**: Unique identifier (e.g., `looptimer-001`)
   - **User Access**: Full Access

### Configure App Information

**Pricing and Availability:**
- **Price**: Free (recommended for first release)
- **Availability**: All countries (or select specific regions)

**App Information:**
- Category: Productivity
- Secondary Category (optional): Utilities
- Age Rating: Complete questionnaire

**App Privacy:**
- Click "Get Started"
- Answer questions about data collection
- For Loop Timer: Select "No, we do not collect data"

## 5. Create Archive & Upload

### Prepare for Archive

1. In Xcode, select **Any iOS Device (arm64)** as destination
2. Ensure scheme is set to **Release** configuration
3. Clean build folder: **Product ‚Üí Clean Build Folder** (Shift+Cmd+K)

### Create Archive

1. **Product ‚Üí Archive**
2. Wait for build to complete (may take several minutes)
3. Organizer window opens automatically

### Validate Archive

Before uploading:
1. Select your archive in Organizer
2. Click **Validate App**
3. Choose **App Store Connect** distribution
4. Select your team
5. Leave default options
6. Click **Validate**
7. Fix any errors or warnings

### Upload to App Store

1. In Organizer, select validated archive
2. Click **Distribute App**
3. Choose **App Store Connect**
4. Choose **Upload**
5. Select distribution options:
   - ‚úì Upload app's symbols (for crash reports)
   - ‚úì Manage version and build number (Xcode will auto-increment)
6. Review signing and provisioning
7. Click **Upload**
8. Wait for upload to complete

### Processing Time

- Upload takes 5-15 minutes depending on connection
- Processing in App Store Connect takes 10-60 minutes
- You'll receive email when processing completes

## 7. Review Process

### Timeline

- **Waiting for Review**: 1-7 days (usually 24-48 hours)
- **In Review**: Few hours to 1 day
- **Total Time**: Typically 24-48 hours from submission

### Review Status

Monitor at: App Store Connect ‚Üí My Apps ‚Üí Loop Timer ‚Üí App Store

**Statuses:**
- **Waiting for Review**: In queue
- **In Review**: Actively being tested
- **Pending Developer Release**: Approved! (you control release)
- **Ready for Sale**: Live on App Store
- **Rejected**: See Resolution Center for reason

### Common Rejection Reasons

**For Timer Apps:**
- Missing app icon
- Crashes or bugs
- Incomplete metadata
- Missing privacy policy
- Poor quality screenshots
- Timer doesn't work as described
- Background audio not functioning

**General Issues:**
- Using placeholder content
- Broken links in description
- App different from screenshots
- Missing required disclosures

### If Rejected

1. Read rejection reason in Resolution Center
2. Fix the issue
3. Increment build number
4. Create new archive and upload
5. Reply in Resolution Center explaining fix
6. Resubmit for review
